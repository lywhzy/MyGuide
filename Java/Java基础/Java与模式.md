### 设计原则

- 开闭原则（ocp) ： 对扩展开放，对修改关闭。 实现的关键在于抽象化
- 里氏替换原则 ： 任何基类出现的地方，都可以用子类来替换
- 依赖倒转原则 ： 要依赖于抽象 而不是依赖于实现
- 接口隔离原则 ： 要尽可能提供较小的接口，而不是提供大的接口
